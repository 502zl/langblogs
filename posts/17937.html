冷却钢包在线监控系项目总结

前端
发布日期:   2021-08-24
文章字数:   1.2k
阅读时长:   4 分

实习项目总结

项目中遇到的问题

  1. 关于文件系统和git默认不区分大小写
    组长在git上修改了我负责组件的文件名大小写，并修改了路由配置，而我本地没有修改，在上传git pull时也没有报冲突，在我上传之后，导致git上关于这个组件拥有两个文件夹，而项目中的路由配置只配置了其中的一个路径，而且在window系统和IOS系统下不区分文件夹大小写，导致git拉取下来两个大小写不同，而字母相同的文件夹会被合并为一个文件夹，并且会以本地的为准，这样，项目第一次能启动，但是那条没在路由配置下的组件就没有热刷新功能。但用Linux系统拉取代码时，能发现两个文件夹存在。
    解决：1. 修改git区分大小写配置（ git config core.ignorecase false ） 2.修改文件名（git mv –force)

  2. 数组响应式问题
    在项目中，我在父组件中定义了一个响应式数组，并把数组中的元素通过props传递给不同的组件，一开始我更新数组是通过数组索引来更新数组（这个不能触发视图的更新，所以会导致父组件中数组已经更新，但子组件中并没有触发更新的问题），但因为我在父组件中使用了一个定时器定时更新组件中的一个响应式对象，这个情境下下会使视图定时被更新，使用导致即使通过索引修改了数组元素，也能使视图更新并传递到子组件；但是，后来的需求改了，，所以移除了定时器，这时候通过索引修改数组就不能导致视图更新，但因为当时移除了这个定时器，就去开发其他功能了，导致这个问题一直被隐藏，在提交代码时，测试功能时才发现，然后通过devtools调试了才发现问题所在
    解决：把通过索引修改数组改变成通过数组的响应式方法来修改数组。

深度思考：Vue 将被侦听的数组的变更方法进行了包裹，包含push()，pop()，shift()，unshift()，splice()，sort()，reverse()这几个方法能响应式修改数组；但数组本质也是对象，而索引也相当于特殊的属性，为什么通过索引更新数组不能触发视图更新呢？？？

看看vue2中的源码：

import { def } from '../util/index'
const arrayProto = Array.prototype
export const arrayMethods = Object.create(arrayProto)
const methodsToPatch = [
  'push',
  'pop',
  'shift',
  'unshift',
  'splice',
  'sort',
  'reverse'
]
methodsToPatch.forEach(function (method) {
  const original = arrayProto[method]
  def(arrayMethods, method, function mutator (...args) {
    const result = original.apply(this, args)
    const ob = this.__ob__
    let inserted
    switch (method) {
      case 'push':
      case 'unshift':
        inserted = args
        break
      case 'splice':
        inserted = args.slice(2)
        break
    }
    if (inserted) ob.observeArray(inserted)
    ob.dep.notify()
    return result
  })
})

可以看到vue的响应式系统中并没有通过属性描述来处理数组响应式，而是对几种会改变原数组的数组方法进行了代理，只有当我们调用数组的这7个方法的时候才会触发视图的更新，这也解释了为什么我们在Vue项目中通过改变数组索引的方式改变数值是不会触发视图更新的。

分析：关于数组有很多变动很大的操作，比如下面：

let arr = [];

arr[1000] = 0;

console.log(arr.length) // 1001

从的上面的操作总结一下，通过Object.defineProperty描述数组为什么不合适

  • 数组和普通对象的区别在于，js中的数组太”多变”了，就像…。比如：arr.length=0，可以瞬间清空一个数组；arr[99]=1有可以瞬间将一个数组的长度变为100，等等骚操作。对于一个普通对象，我们一般只会改变key对应的value值，而不会连key都改变了,当key改变时我们只需要在拦截器中将重新赋值的value使用Object.defineProperty给它加上拦截器就行了。但是数组就不一样了，因为其多变，如果使用Object.defineProperty()去添加拦截，我们需要重新将整个数组对象的所有key递归的加上拦截器并且我们还要穷举每一种数组变化的可能，这样势必就会带来性能开销问题，如果数组对象中存的数据量大而且操作频繁的时候，这就是一个大问题。
  • 数组在应用中经常会被操作，但是通常’push’,’pop’,’shift’,’unshift’,’splice’,’sort’,’reverse’这7中方法，基本上也能达到我们的目的。因此如果为了使数组具备通过value的改变就能触发视图更新的能力，而使用Object.defineProperty就显得代价太大了。
