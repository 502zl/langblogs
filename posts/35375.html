防抖和节流

前端
发布日期:   2021-07-25
文章字数:   627
阅读时长:   2 分

引入背景

防抖和节流是针对响应跟不上触发频率这类问题的两种解决方案。 在给 DOM 绑定事件时，有些事件我们是无法控制触发频率的。 如鼠标移动事件 onmousemove, 滚动滚动条事件 onscroll，窗口大小改变事件 onresize，瞬间的操作都会导致这些事件会被高频触发。 如果事件的回调函数较为复杂，就会导致响应跟不上触发，出现页面卡顿，假死现象。 在实时检查输入时，如果我们绑定 onkeyup 事件发请求去服务端检查，用户输入过程中，事件的触发频率也会很高，会导致大量的请求发出，响应速度会大大跟不上触发。
针对此类快速连续触发和不可控的高频触发问题，debounce 和 throttling 给出了两种解决策略

防抖（debounce)

高频率触发的事件，在指定的单位时间内，只响应最后一次，如果在指定的时间在触发，则重新计算时间(后面触发的事件执行，替代了前面的事件)
总结：一定时间内，只执行最后一次
特点：如果事件在规定的时间间隔内被不断的触发，则调用方法会被不断的延迟

应用：

  1. search搜索联想，用户在不断输入值时，用防抖来节约请求资源。
  2. 验证用户输入是否规范时，使用防抖来减少验证方法的多次触发
  3. window触发resize的时候，不断的调整浏览器窗口大小会不断的触发这个事件，用防抖来让其只触发一次

函数封装：


// callback 为定时器时间到了，需要执行的回调
function debounce(callback, delay) {
  let timer = null;
  return function() {
    clearTimeout(timer);
    timer = setTimeout(function() {
      callback.apply(this, arguments)
    }, delay);
  }
}

let debounceFn = debounce(callback, delay);

节流(throttling)

高频率触发的事件,在指定的单位时间内，只响应第一次(前面触发的执行前，忽略后面的事件)
总结：一定时间内，只执行一次
特点：会稀释函数的执行频率

应用：

  1. 鼠标不断点击触发，mousedown(单位时间内只触发一次)
  2. 监听滚动事件，比如是否滑到底部自动加载更多，用throttle来判断

函数封装

function throttling(callback, delay) {
  // 记录是否已经开始计时
  let timer;
  return function() {
    if(timer) return;
    timer = setTimeout(() =>{
      callback.apply(this, arguments)
      clearTimeout(timer)
    }, delay)
  }
}

let throttlingFn = throttling();
