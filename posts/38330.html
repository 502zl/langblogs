知识基垫

优雅的工厂函数——构造函数

// 一个构造函数
function Person(name) {
  this.name = name
  this.sayHello = function() {
    console.log("你好，我是",this.name)
  }
}

// 看下面两种方式创建实例的区别，就能体会到new做的事了
// const zf = Person("张飞")  // undefined
const zf = new Person('张飞')
zf.sayHello() // 你好， 我是张飞

// 创建一个实例：
// 这里要注意使用new创建实例，构造函数内部会做的事；
//    1. 创建一个新的空对象 （let instance = {} )
//    2. 把新对象的原型指向构造函数的原型对象上（instance.__proto__ = Person.prototype）
//    3. 将构造函数的作用域赋给新对象（this = instance）
//    4. 执行构造函数中的代码 (this.name = name ...)
//    5. 返回实例 (return this)

// 每个对象都有一个constructor属性指向它的构造函数，使用字面量直接声明的对象，其构造函数为Object()
console.log(zf.constructor === Person)  // true

构造函数的弊端

// 内存问题
// 当我们需要给实例添加一些固定的属性时
// 值使用构造函数的话1，会这样写
function Person(name) {
  this.name = name
  this.type = "三国"
  this.sayHello = function() {
    console.log("你好，我是", this.name)
  }
}

const zf = new Person("张飞")
const lb = new Person("刘备")
// 这样写的话，我们每使用new去创建一个实例，this.type = "三国" 和 this.sayHello =... 都会被执行一遍，这样重复的动作，会造成内存浪费

// 普通的解决方式：将需要共享的方法放在构造函数外，然后给它的属性赋值一个应用就行
let sayHello = function() {
  console.log("你好，我是", this.name)
}
function Person1(name) {
  this.name = name
  this.type = "三国"
  this.sayHello = function() {
    console.log("你好，我是", this.name)
  }
}

// 注意, **在构造函数外添加的新属性实例访问不到滴**
Person1.age = 18;
const gy = new Person("关羽")
console.log(gy.age) // undefined

// 思考：这样做会不会造成其他的问题呢？
// 大聪明，肯定会啊，当你有多个构造函数，但是你又想用同一个方法名时，就会造成命名冲突，当然这个也可以解决，可以在外边再套一层对象，但是这样写代码就一点都不优雅了

所以原型就来解决这个问题，能让我们js程序猿在创建对象时显得节俭而不失大雅，并向面向对象看齐。
虽然现实生活中，想面向对象编程并没有那么容易。

原型的导入（原型模式）

prototype原型对象可以理解为一个对象模版

Javascript 规定，任何一个函数在创建之后，便会拥有一个prototype属性，这个属性指向函数的原型对象，而这个对象的所有属性和方法，都会被构造函数的实例所继承。

原型对象的两种导出：

隐式原型（__proto__） 显示原型（prototype）

  • 所有引用类型都有一个__proto__(隐式原型)属性，属性值是一个普通的对象

  • 所有函数都有一个prototype(原型)属性，属性值是一个普通的对象

  • 所有引用类型的__proto__属性指向它构造函数的prototype

  • 所有引用类型的__proto__都是指向Function的prototype

  • 构造函数new出来的对象__proto__指向构造函数的prototype

  • 非构造函数实例化出的对象或者对象的prototype的__proto__指向Object的prototype

  • Object的prototype指向null（最顶级的）

(前几天做一个测试，一直在终端打印对象的__proto__属性，一直报undefined，今天总结，突然发现属性名开头和结尾都是两条小横线，坑死了。。。。)

图解

上代码

// 构造函数
function Person(name) {
  this.name = name;
}

// 将共享的属性放在原型上去声明
Person.prototype.myTime = function() {
  console.log("我是三国英雄")
}

Person.prototype.sayHello = function(){
  console.log("你好, 我是", this.name)
}


const p1 =  new Person('张飞') // ok
const p2 =  new Person('刘备') // ok
p1.myTime() // 我是三国英雄
p2.myTime() // 我是三国英雄

console.log(typeof Person.prototype) // object
console.log(Person.prototype.constructor === Person) // true
console.log(Person.prototype === p1.__proto__) // true
console.log(p1.__proto__ === p2.__proto__) // true

console.log(People.prototype.__proto__.constructor) // Object()
console.log(Array.prototype.__proto__.constructor)  // Object()
console.log(Object.prototype.__proto__) // null

原型对象总结：
所有的 JavaScript 对象都会从一个 prototype（原型对象）中继承属性和方法：
(所有对象都由函数构造而来，所有的函数都有一个原型对象)
Date 对象从 Date.prototype 继承。
Array 对象从 Array.prototype 继承。
Person 对象从 Person.prototype 继承。
所有 JavaScript 中的对象都是位于原型链顶端的 Object() 的实例

更为优雅的使用原型：

// 构造函数
function Person(name) {
  this.name = name;
}

// 把所有共享的属性一次性放进原型对象中声明
Person.prototype = {
  myTime: function() {
    console.log("我是三国英雄")
  },
  sayHello: function() {
    console.log("你好, 我是", this.name)
  }
}

console.log(Person.prototype)
/*
原始的写法， Person.prototype
{myTime: ƒ, sayHello: ƒ, constructor: ƒ}

上面的写法之后
现在的 Person.prototype
{myTime: ƒ, sayHello: ƒ}

** 发现构造器丢失
*/

但是这样写的话，相当于重写了Person.prototype，导致它原来拥有的constructor被丢失

所以，需要把构造器带上

// 构造函数
function Person(name) {
  this.name = name;
}

// 把所有共享的属性一次性放进原型对象中声明
Person.prototype = {
  constructor: Person,
  myTime: function() {
    console.log("我是三国英雄")
  },
  sayHello: function() {
    console.log("你好, 我是", this.name)
  }
}

这样就真正优雅了。

原型链

当我们创建一个实例后，肯定是要使用这个实例的属性去完成某些操作，当然，使用我们自己写的构造函数中的属性，一目了然皆可以找到，那么你应该知道每个实例都有一个toString()方法，但我们在构造函数中并没有定义，那么javascript是怎样找到我们要用到的属性和方法呢？？？

这里就是原型链的概念：
JavaScript 对象有一个指向一个原型对象的链。
当试图访问一个对象的属性时，它不仅仅在该对象上搜寻，还会搜寻该对象的原型，以及该对象的原型的原型，
依次层层向上搜索，直到找到一个名字匹配的属性或到达原型链的末尾。
这样一层一层向上查找就会形成一个链式结构，即原型链咯。

你不会觉得难以理解吧。。。

那我再简化一点咯

  1. 现在自己身上找
  2. 自己身上找不到，就去父亲（实例的__proto__）身上找，如果没找到，继续找父亲的父亲
  3. 直到找到Object.prototype.__proto__（null），都没有找到，就会返回undefined了

javascript 中的继承

看到继承二字，写过c++，java的同学并不陌生，因为c++，java等一系列面向对象的编程语言，都是建立在对象（class）之上的，而他们具有继承，封装，重载这三大特效；但实际上，javascript起初是没有类这个概念的，但随着它越做越大，就开始往面向对象靠近，模拟类的概念，以至于ES6中新添了可以用class模板类，为什么要这么做？我的浅显理解就是面向对象这个思想在编程世界里面，处理实际问题拥有超乎想象的好处，往数据共享，代码复用，还有封装，内存方面考虑。

继承的概念就不说了，反正我的博客也只有我一人看，哈哈哈。

直接上代码了。

继承

  1. 使用call实现继承
function Person(name) {
  this.name = name
}

Person.prototype.sayHello = function() {
  console.log("你好，我是", this.name)
}

function Student(name, role, age) {
  Person.call(this, name)
  this.role = role
  this.age = age
}

const stu1 = new Student("张飞", "学生", 18)
console.log(stu1) // Student {name: "张飞", role: "学生", age: 18}
console.log(Student.prototype.__proto__.constructor)  // ƒ Object()

分析：
从stu1的打印结果，我们可以看到，使用这种改变函数执行上下文的方式实现继承，只能继承父类构造函数中的属性，而父类原型对象中的属性，不能被继承。
所以，需要把父类的原型链赋给子类，让子类在父类原型链的下方，即要让子类的原型指向父类

  1. 使用原型实现继承
function Person(name) {
  this.name = name
}

Person.prototype.sayHello = function() {
  console.log("你好，我是", this.name)
}

function Student(role, age) {
  this.role = role
  this.age = age
}

Student.prototype = new Person("张飞")
Student.prototype.constructor = Student // 上一行，Student的原型被覆盖，constructor属性丢失，这里要重新补上，上面有案例说过的

const stu1 = new Student("学生", 19) 
const stu2 = new Student("学生", 20) 

console.log(stu1)  // Student {role: "学生", age: 19} [[Prototype]]: Person
console.log(stu2)  // Student {role: "学生", age: 20} [[Prototype]]: Person

// 注意这里！！！
stu1.sayHello() // "你好，我是 张飞"
stu2.sayHello() // "你好，我是 张飞"

console.log(Student.prototype.__proto__.constructor) // ƒ Person(name)

分析：
从代码执行结果，可以看出，Student确实在Person原型链的下方，所以stu1能够访问到Person中的属性
但是，从stu1和stu2访问父类的属性时，可以看到，他们的值是一样的，即继承过来的不止属性，还继承过来了属性值，这导致创建子类实例时，他们必须共用相同的父类的属性，我们还需要重新初始化，这不是扯淡吗。。。

这时，我们发现，把上面两种结合起来，有没有瞬间明朗了
所以，组合继承来咯

  1. 组合继承

在子类内部借用父类的执行上下文，并在外部，“嫁接”到父类的原型链上去

上代码：

function Person(name) {
  this.name = name
}

Person.prototype.sayHello = function() {
  console.log("你好，我是", this.name)
}

function Student(name, role, age) {
  Person.call(this, name)
  this.role = role
  this.age = age
}

Student.prototype = new Person()
Student.prototype.constructor = Student

const stu1 = new Student("张飞" ,"学生", 19) 
const stu2 = new Student("刘备" ,"学生", 20) 

console.log(stu1)  // Student {name: "张飞", role: "学生", age: 19} [[Prototype]]: Person
console.log(stu2)  // Student {name: "刘备", role: "学生", age: 20} [[Prototype]]: Person

// 注意这里！！！
stu1.sayHello() // "你好，我是 张飞"
stu2.sayHello() // "你好，我是 刘备"

console.log(Student.prototype.__proto__.constructor) // ƒ Person(name)

以上的继承尚且和java里面的继承有点相似
但，下面，会使用一种全新得继承方式，个人感觉不太优雅。

  1. 拷贝继承

就像它的名字一样，就是把一个父类的属性直接复制给子类
直接上代码吧

function Person(name) {
  this.name = name
}

Person.prototype.type = "三国"
Person.prototype.sayHello = function() {
  console.log("你好，我来自", this.type)
}

function Student(role, age) {
  this.role = role
  this.age = age
}
let parent = Person.prototype
let child = Student.prototype

for(let arr in parent) {
  child[arr] = parent[arr]
}

// 我们看一看Student的原型链
console.log(Student) // f Student(role, age){...}
console.log(Student.prototype.constructor)  // f Student(role, age){...}
console.log(Student.prototype.__proto__) // 由Object() 创建的一个实例
console.log(Student.prototype.__proto__.constructor) // f Object()

const stu1 = new Student("学生", 19) 
const stu2 = new Student("学生", 20) 

console.log(stu1)  // Student { role: "学生", age: 19} [[Prototype]]: Object
console.log(stu2)  // Student { role: "学生", age: 20} [[Prototype]]: Object

stu1.sayHello() // "你好，我来自 三国"
stu2.sayHello() // "你好，我来自 三国"

Student.prototype.sayHello = function() {
  console.log("曹贼，纳命来！！！！")
}

Person.prototype.sayHello() // 你好，我来自 三国
Student.prototype.sayHello() // 曹贼，纳命来！！！！

分析：
从上面的执行结果可以看出，这个拷贝继承并没有实现真正的继承啊，只是拷贝了属性而已，还要注意这个拷贝为浅拷贝，所以这个方法，并不优雅。。。。

再看看下面这种方式

  1. 直接继承

上代码：

function Person(){};
Person.prototype.name = "张飞";
Person.prototype.age = 18;

function Student(){};
Student.prototype = Person.prototype;

console.dir(Student);
console.dir(Person);

// 没有下面这一句时
// Person的原型和Student原型的age都为18， 但有了下面这一行之后，他们俩的原型都变为 25！！！这就是直接赋值的弊端
// Student.prototype.age=25;

分析：
优点 : 效率比较高
缺点 : 因为相当于是个传址过程 所以修改Student原型的属性 Person原型的属性也会被更改

所以有了下面的优化

  1. 使用空对象作为介质实现继承（寄生组合继承）
function Person() {}
Person.prototype.name = "张飞";
Person.prototype.age = 11;

function Student() {}
const F = function () {};
F.prototype = Person.prototype;

Student.prototype = new F();
Student.prototype.constructor = Student;

Student.prototype.age = 25;

console.dir(Person);  // ƒ Person() 原型的age为11
console.dir(Student); // ƒ Student() 原型的age为25

// 看一看Student的原型链
console.dir(Student.prototype.constructor) // f Student()
console.dir(Student.prototype.__proto__) // 由Object() 创建的一个实例，其constructor为Person()
console.dir(Student.prototype.__proto__.constructor) // f Person()

分析：
很优雅，哈哈哈哈。
